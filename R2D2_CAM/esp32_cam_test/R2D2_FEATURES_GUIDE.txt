⚠️ FEATURE 2: MOTION DETECTION SYSTEM
================================================================================

WHAT IT DOES:
- Automatically detects movement in the camera view
- Compares each frame with the previous frame
- Calculates motion percentage
- Triggers visual and serial monitor alerts
- No extra hardware needed!

HOW IT WORKS:
1. Camera captures frames continuously
2. Each frame is compared to the previous frame
3. Pixel differences are calculated
4. Motion percentage is computed
5. If motion exceeds threshold (default: 15%), alert triggers

MOTION THRESHOLD SETTINGS:
- Default: 15% motion required to trigger
- Lower = more sensitive (10-12%)
- Higher = less sensitive (20-25%)
- To adjust: Change "motionThreshold" value in code (line ~25)

INDICATORS:
🔵 Scanning... - No motion detected
🔴 ⚠ MOTION DETECTED - Motion above threshold

REAL-TIME STATISTICS:
- Current motion level percentage
- Threshold setting
- Detection status

SERIAL MONITOR OUTPUT:
When motion is detected, you'll see:
⚠ MOTION DETECTED: 23.4%

USE CASES FOR R2D2:
- Security droid mode
- Automatic recording trigger
- Motor movement trigger (future integration)
- Area monitoring

TROUBLESHOOTING:
Q: Constant false detections?
A: Increase threshold to 20-25%

Q: Not detecting motion?
A: Decrease threshold to 10-12%

Q: Detection too slow?
A: Adjust sampling rate in detectMotion() function

📸 FEATURE 3: PHOTO CAPTURE GALLERY
================================================================================

WHAT IT DOES:
- Capture high-resolution photos from live stream
- Stores photos in ESP32-CAM's flash memory (SPIFFS)
- View all captured photos in a gallery
- Download photos to your computer
- Delete unwanted photos
- Persistent storage (survives power cycle)

CAPTURING PHOTOS:
1. Click the "📸 Capture Photo" button on main page
2. Photo is instantly saved to flash memory
3. Confirmation alert shows filename and size
4. Photo counter updates automatically

PHOTO GALLERY FEATURES:
📁 Grid layout of all captured photos
📊 Photo info (filename, size in KB)
⬇️ Download button for each photo
✕ Delete button to remove photos
🔄 Auto-refresh every 5 seconds
📱 Responsive design for mobile viewing

VIEWING GALLERY:
1. Click "🖼 Photo Gallery" button on main page
2. See all your captured photos
3. Click download to save to computer
4. Click delete to remove from ESP32

STORAGE DETAILS:
- File format: JPEG
- File naming: photo_001.jpg, photo_002.jpg, etc.
- Storage location: ESP32-CAM flash memory (SPIFFS)
- Storage capacity: ~3MB available (approx. 50-100 photos)
- Persistent: Photos survive power cycling!

DOWNLOADING PHOTOS:
Method 1 (Gallery):
1. Open gallery
2. Click "⬇ Download" on any photo
3. Photo downloads to your computer

Method 2 (Direct URL):
1. Note the photo filename
2. Go to: http://192.168.4.1/download?file=/photo_001.jpg
3. Photo downloads directly

DELETING PHOTOS:
1. Open gallery
2. Click "✕ Delete" on the photo
3. Confirm deletion
4. Photo is permanently removed

MAXIMUM PHOTOS:
- ESP32-CAM has ~3MB SPIFFS partition
- Each photo is ~20-40KB
- Can store approximately 75-150 photos
- When full, you'll need to delete some photos

🔧 ADVANCED CUSTOMIZATION
================================================================================

CHANGE MOTION SENSITIVITY:
Line 25 in the code:
float motionThreshold = 15.0;  // Change this value (5.0 - 50.0)

CHANGE WiFi NAME:
Line 16 in the code:
const char* ap_ssid = "R2D2-CAM";  // Change network name

CHANGE WiFi PASSWORD:
Line 17 in the code:
const char* ap_password = "12345678";  // Must be 8+ characters

ADJUST SCANLINE INTENSITY:
In the CSS, modify:
rgba(0,0,0,0.15) → Increase for darker lines (0.2, 0.3)
rgba(0,0,0,0.15) → Decrease for lighter lines (0.1, 0.05)

ADJUST HOLOGRAM COLOR:
In the CSS #cam filter, modify:
hue-rotate(180deg) → Change color tint (try 120deg for green, 240deg for purple)

CHANGE FRAME RATE:
In JavaScript, modify:
setInterval(refreshImage, 30); → 30 = 33fps, 50 = 20fps, 100 = 10fps

================================================================================
⚠️ TROUBLESHOOTING
================================================================================

PROBLEM: Photos not saving
SOLUTION: Check Serial Monitor for SPIFFS errors, may need to reformat

PROBLEM: Gallery shows "Loading..."
SOLUTION: Refresh the page, check Serial Monitor for errors

PROBLEM: Motion detection always triggering
SOLUTION: Increase motionThreshold value

PROBLEM: Hologram effect too strong
SOLUTION: Adjust CSS filters in handleRoot() function

PROBLEM: Can't connect to WiFi
SOLUTION: Check that ESP32-CAM is powered properly, wait 10 seconds after boot

PROBLEM: Stream is slow/laggy
SOLUTION: Reduce frame rate, lower resolution, check WiFi signal
